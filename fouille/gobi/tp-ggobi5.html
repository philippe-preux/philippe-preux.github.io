Objectifs du TP

L'objectif est de manipuler la MDS dans ggobi.

Il y a deux utilisations possibles de la MDS dans ggobi :

  1. analyse d'une matrice de dissimilarités
  2. représentation d'un graphe

Durant ce TP, on s'intéresse uniquement à la représentation d'un graphe. Un graphe est composé de nœuds et d'arcs entre ces nœuds. L'analyse de dissimilarité sera vue dans le TP suivant.
Pour représenter un graphe sous la forme d'un fichier xml pour ggobi, il faut que ce fichier soit composé de deux parties (deux jeux de données), le premier représentant les nœuds, le second les arcs entre ces nœuds. Chaque nœud et chaque arc peut par ailleurs posséder des attributs, comme d'habitude. En cliquant ici, vous obtenez un moule de fichier xml pour créer un graphe au format attendu apr ggobi.

À faire
Un exemple simple : un cube
  • Créér un fichier cube.xml qui représente un cube. Vous inspirez de ce fichier et de la documentation.
  • qu'espérez-vous trouvez par une MDS ?
  • faites-en la MDS.

Ensuite :

  1. ouvrir ce fichier avec ggobi ;
  2. accéder à l'outil MDS : dans le menu « Tools », choisir « ggvis (MDS) ... ». Une fenêtre s'ouvre pour cet outil.
  3. dans l'onglet « Task », cliquer sur « Graph Layout » : on veut représenter un graphe et c'est comme cela qu'on le spécifie à ggobi ;
  4. décliquer le bouton « Complete graph distances » (il doit être de la même couleur, gris clair, que celui au dessus) ;
  5. dans l'onglet « Run », la dimension sélectionnée est 3. Cliquez sur « Run MDS » ; une fenêtre s'ouvre ; dans le menu « Reset », sélectionnez « Scramble layout » pour mélanger les nœuds ; vous devez ensuite voir :
    • les points bougent
    • la fonction Stress est représentée est décroît (initialement)
  6. manipuler les petits menus situés en dessous de la fonction stress. Vous pouvez ainsi sélectionner les différents types de MDS ;
  7. dans le menu « View », sélectionner le Shepard Plot qui est un histogramme des distances entre les nœuds ;
  8. faites le tour de toutes possibilités. Jetez un coup d'œil à la documentation. Rappelez-vous que la MDS fournit un optimum local ; n'hésitez pas à re-mélanger les points pour, éventuellement, atteindre un autre optimum ;
  9. dans la fenêtre graphique, dans le menu « Edges », faites apparaître les arcs. Que constatez-vous ? Êtes-vous satisfaits du résultat ?

Réseau de relations

On considère un réseau de relations entre 16 personnes, chacune étant désignée par une lettre, tel qu'il est indiqué dans le tableau suivant :

abcde fghij klmnop
a xxx
b x xx
c xx x
d xxx x x
e x xxx x
f xx xxx
g x xx
h xx x
i xxx
j x xxx
k x xx
l x x x
m x x xx xx
n x x
o x x
p x

Une x indique que que les deux personnes se connaissent. Un vide indique qu'elles ne se connaissent pas. La relation est symétrique.

  • créer le fichier reseauDeRelations.xml ;
  • en faire une MDS. Trouvez-vous des groupes de gens qui se connaissent comme cela apparaît dans cette table ?
