Format xml

xml est un format de fichiers qui a pour vocation à être un standard. Il n'est pas inutile de le connaître un peu, même si ce n'est absolument pas le but du TP, et encore moins celui du cours.

Dans ce genre d'entreprise, le plus simple est de regarder un fichier existant pour créer un nouveau fichier. Par exemple, vous pouvez regarder le fichier olive.xml que l'on a étudié avec ggobi.

Principe général de la structure d'un fichier xml : c'est très simple. Les choses entre < et > sont des noms de balise. Comme leur nom l'indique, ce sont des repères dans le fichier xml, qui en indique les différents champs. Quasiment systématiquement, après le caractère <, on trouve une chaîne de caractère, par exemple <toto> : cette balise ouvre un champ dénommé toto ; ce champ est obligatoirement terminé par une balise fermante </toto>.

Je suppose à ce stade que vous avez ouvert le fichier olive.xml avec un éditeur de textes quelconque et que vous l'avez devant les yeux. Je commente les différentes lignes :

  • ligne 1 : à copier tel quel dans votre fichier. Cette ligne indique que le fichier contient du xml, version 1.0 ;
  • ligne 2 : à copier tel quel dans votre fichier. Cette ligne indique la structure du fichier xml qui suit ;
  • ligne 4 : à copier tel quel dans votre fichier. Cette ligne indique que le fichier ne contient qu'un seul de données ;
  • ligne 5 : à copier tel quel dans votre fichier. Cette ligne indique des détails concernant la manière de visualiser les points ;
  • ligne 7 : indique le nom du jeu de données : mettez-y ce qu'il faut ;
  • ligne 8 à 19 : entre les balises description, vous indiquez ce que contient le jeu de données ;
  • ligne 20 : indique le nombre d'attributs : adaptez ! ;
  • ligne 21 à 27 : ces lignes décrivent un attribut nominal :adaptez à votre cas. Cherchez à comprendre ce que signifie ces informations ;
  • ligne 28 à 40 : idem pour un autre attribut nominal ;
  • ligne 41 à 48 : chaque ligne décrit un attribut numérique. Adaptez à votre cas !
  • ligne 49 : à copier tel quel dans votre fichier. Cette ligne indique que c'est la fin de la définition des attributs ;
  • ligne 50 : une ligne qui indique le nombre de données ;
  • ligne 51 à 53, jusque la ligne 1766, par groupe de 3 lignes : chaque groupe de trois lignes spécifie la valeur de tous les attributs d'une donnée ;
  • ligne 1767 : à copier tel quel dans votre fichier. Cette ligne indique la fin des données ;
  • ligne 1768 : à copier tel quel dans votre fichier. Cette ligne indique la fin du jeu de données ;
  • ligne 1769 : à copier tel quel dans votre fichier. Cette ligne indique la fin du fichier spécifiant le jeu de données.

Créez donc un fichier tennis.xml en suivant ces indications. Faites en sorte que l'on puisse facilement identifier chaque donnée dans ggobi.
