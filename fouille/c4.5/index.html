Les 4 premiers exercices sont liés : les 3 premiers permettent de découvrir le logiciel c4.5 sur un exemple simple. L'exercice 4 consiste à effectuer quelques expériences supplémentaires sur ce jeu de données donnant quelques pistes pour analyser plus finement les performances de l'arbre de décision obtenu. L'exercice 5 consiste à travailler de manière plus autonome sur un autre jeu d'exemples.

Objectifs du TP

Manipuler le logiciel de construction d'arbres de décision C4.5.
Remarque : c4.5 est le fruit d'une recherche académique et résulte du travail de plusieurs années. Il est gratuit pour les applications pédagogiques et dans le cadre d'activités de recherche ; son utilisation dans un autre contexte n'est pas gratuite.

Exercice 1 : construction d'un arbre de décision

On considère le jeu de données se trouvant dans les fichiers dont le nom commence par tennis. Ce jeu de données est décrit dans deux fichiers :

  • tennis.names qui décrit les attributs et leurs valeurs possibles : la première ligne indique les valeurs possibles de l'attribut classe ; les lignes suivantes donnent pour chaque attribut, les valeurs possibles séparées par une virgule si cet ensemble est discret, le mot-clé continuous si l'ensemble est continu ;
  • tennis.data qui contient les valeurs des attributs des individus du jeu de données. Chaque ligne correspond à un individu ; la valeur de ses attributs est donnée en siuvant l'ordre dans lequel sont listés les attributs dans le fichier tennis.names ; la dernière valeur est celle de la classe de l'individu.

Récupérer et sauvegarder ces deux fichiers en cliquant dessus ; les mettre dans le répertoire R8/Data. Analyser le contenu de ces deux fichiers.
Note à l'égard des grands débutants sous Linux : pour voir le contenu d'un fichier, le moyen le plus simple est d'utiliser la commande cat suivie du nom du fichier, par exemple cat tennis.names.

Pour construire un arbre de décision à partir du jeu de données tennis, on tape la commande :
c4.5 -f tennis
À la suite de cette commande, un certain nombre de choses sont affichées ; on trouve en particulier :

  • le fait que 14 individus décrits chacun par 4 attributs ont été utilisés ;
  • l'arbre de décision qui a été construit sous une forme vaguement graphique ;
  • l'évaluation sur les individus du jeu de données de cet arbre de décision avant et après élagage de l'arbre (élaguer = to prune en anglais).

Repérer toutes ces informations dans ce que la commande a affiché à l'écran. En particulier, on pourra dessiner l'arbre produit par c4.5 pour mieux le visualiser et repérer le nombre d'erreurs de classification commises.

Exercice 2 : classification de nouveaux exemples

Une fois l'arbre de décision construit, on peut l'utiliser pour classer de nouvelles instances. Pour cela, on utilise la commande :
consult -f tennis
Celle-ci demande la valeur des attributs d'une nouvelle individu en parcourant l'arbre de décision. C'est donc la valeur de l'attribut testé en racine de l'arbre qui est demandée d'abord. Ensuite, en fonction de cette valeur, l'arbre est parcouru et la valeur de nouveaux attributs est demandée jusqu'à ce que l'instance puisse être classée. Finalement, l'individu est classé et la confiance dans la classification sont indiqués, la confiance étant indiquée par CF = ...
Classer :

  • un individu dont les attributs sont : outlook = sunny, temperature = hot, humidity = normal, windy = false ;
  • un individu dont les attributs sont : outlook = overcast, temperature = warm, humidity = high, windy = true ;
  • un individu dont les attributs sont : outlook = rain, temperature = cool, humidity = high, windy = true.

Que constatez-vous ?

On peut aussi ne pas spécifier la valeur d'un attribut en tapant '?' lorsque sa valeur est demandée. Par exemple, classer un individu dont on ne connaît pas la valeur de outlook, temperature vaut warm, humidity vaut high et windy est true.
Interpréter le résultat.

Classer un individu dont on ne connaît la valeur d'aucun attribut.
Interpréter le résultat.

Exercice 4 : quelques expériences

On reprend le jeu de données tennis parce que c'est le plus simple. On veut observer la dégradation des performances de l'arbre de décision en fonction du nombre d'exemples fournis pour l'apprentissage. Pour cela, on va procéder comme suit : en partant du jeu de données initial, on va constituer des jeux d'apprentissage de plus en plus pauvres, c'est-à-dire contenant de moins en moins d'exemples. À chaque fois que l'on aura retiré un exemple du jeu d'apprentissage, on reconstruira un arbre de décision avec ce nouveau jeu de données appauvri et on notera le taux d'erreurs de classification effectuées sur l'ensemble du jeu de données initial.

  • à partir du fichier tennis.data, composez 13 fichiers (appelez-les tennis-1.data, tennis-2.data, ... tennis-13.data par exemple) en retirant à chaque fois une ligne (= un exemple) du fichier précédent. Vous pouvez retirer la première ligne à chaque fois, ou retirer les lignes au hasard ;
  • à chacun de ces 13 fichiers doit être associé un fichier .names : vous pouvez obtenir le résultat en tapant 13 instructions ressemblant à la suivante :
    ln -s tennis.names tennis-1.names
  • à chacun de ces 13 fichiers doit être associé un fichier .test qui contient les exemples sur lesquels il faut évaluer les performances de l'arbre. Comme on l'a dit plus haut, c'est l'ensemble d'exemples initial. Vous pouvez obtenir le résultat en tapant 13 instructions telles que : 
      ln -s tennis.data tennis-1.test
Arrivé à ce stade, vous disposez de tous les fichiers nécessaires. Nous allons conduire l'expérimentation proprement dite.
  • reste donc à construire les arbres de décision sur ces 13 jeux de données et à évaluer les performances de classification sur le jeu de données initial. Pour le jeu tennis-1, vous obtiendrez le résultat voulu en tapant la commande :
    c4.5 -f tennis-1 -u L'option -u indique que l'arbre construit doit être évalué sur le jeu de données se trouvant dans le fichier tennis-1.test.
    En observant le résultat de cette commande, vous voyez que l'information qui vous intéresse (taux d'erreur) se trouve sur la huitième ligne en partant de la fin. Cette ligne sera extraite automatiquement en tapant la commande suivante :
    c4.5 -f tennis-1 -u | tail -8 | head -1 Remarque pour le linuxien débutant : dans cette commande, la notation | tail -8 indique qu'il faut ne considérer que les 8 dernières lignes générées par la commande précédant la |, soit c4.5 -f tennis-1 -u. De même, | head -1 indique qu'il ne faut prendre que la première ligne de ce qui est généré par ce qui précède, c'est-à-dire, la première ligne parmi les 8 dernières lignes générées par c4.5 -f tennis-1 -u. Le tour est joué...
    Cette ligne met en oeuvre un mécanisme puissant des commandes Linux : le pipeline : ce qui est généré par une commande (à gauche d'une |) est envoyé vers une autre commande (à droite d'une |). Ce qui est réellement magnifique, c'est que l'on peut itérer ce procédé autant de fois que l'on veut : on met bout à bout des commandes, autrement dit, on les pipeline. Ici, on a pipeliné 3 commandes.
    Cette ligne ainsi sélectionnée contient dans l'ordre :
    • la taille de l'arbre avant élagage
    • le nombre d'exemples mal classés par l'arbre non élagué
    • la proportion que cela représente parmi l'ensemble des exemples de tests
    • la taille de l'arbre après élagage
    • le nombre d'exemples mal classés par l'arbre élagué
    • la proportion que cela représente parmi l'ensemble des exemples de tests
    • une estimation de l'erreur réalisé par l'arbre élagué sur un jeu d'exemples quelconque
    On peut par exemple tracer le taux d'erreur de l'arbre non élagué en fonction de la taille du jeu d'apprentissage.
    Pour cela :
    • pour chaque jeu de données, placer dans un fichier un ensemble de lignes contenant chacune : le nombre d'exemples dans le jeu d'apprentissage, les informations fournies par c4.5 sur la huitième ligne avant la fin (retirer les parenthèses, % et autres << : remplacer les par des espaces). Appelons ce fichier tennis.resultats ;
    • on peut utiliser gnuplot pour tracer un joli graphe comme suit : (ou un autre logiciel, R par exemple) 
        gnuplot
  set data style lines
  plot "tennis.resultats" using 1:4
      Cet ensemble de commandes trace la courbe voulue.
      Vous pouvez améliorer votre graphe en ajoutant des légendes comme suit : 
        set xlabel "Nombre d'exemples dans le jeu d'apprentissage"
  set ylabel "Taux d'erreurs"
  set title "Influence du nombre d'exemples d'apprentissage sur le taux d'erreurs"
  plot "tennis.resultats" using 1:4 title "avant élagage"
      Vous devriez obtenir quelque chose qui ressemble à cela :
    • vous pouvez aussi obtenir sur le même graphe la variation du taux d'erreur avant et après élagage comme suit : 
        plot "tennis.resultats" using 1:4 title "avant élagage", "tennis.resultats" using 1:7 title "après élagage"
    • vous pouvez aussi obtenir sur le même graphe la variation du taux d'erreur avant, après élagage et estimation de l'erreur comme suit : 
        plot "tennis.resultats" using 1:4 title "avant élagage", "tennis.resultats" using 1:7 title "après élagage", "tennis.resultats" using 1:8 title "estimation"
      Il ne vous aura pas échappé que l'on indique les colonnes du fichier de données à utiliser par l'option using, et que l'on donne un titre à une courbe par l'option title.
      Pour quitter l'environnement gnuplot, taper la commande quit.
Vous en savez maintenant suffisamment pour continuer l'analyse expérimentale du comportement de c4.5 sur un jeu de données.

Par exemple, vous pouvez tracer les courbes indiquant le nombre de faux positifs et de faux négatifs en fonction du nombre d'exemples dans le jeu d'apprentissage.
