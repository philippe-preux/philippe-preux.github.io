Mise en pratique

En R

Historiquement (début des années 2000), j'ai utilisé R comme environnement logiciel support pour mes cours de fouille de données.

Ci-dessous, des pointeurs vers des sujets de TP abordant les différents points de mon cours :

  • découverte de R ; voir aussi généralités sur R
  • classification supervisée par arbres de décision
  • exercice de probabilités
  • segmentation de données (classification non supervisée) :
    • segmentation de données par les k-moyennes ;
    • segmentation hiérarchique ascendante de données ;
    • segmentation par les k-médianes ;
    • exploration visuelle des données
  • préparation des données
  • réduction de dimension (ACP, MDS)
  • visualisation de données avec gobi :
    • découverte de Gobi,
    • une ACP dans Gobi,
    • représentation d'un graphe,
    • analyse de dissmilarité.
    Gobi est ici utilisé indépendemment de R. Le paquet Rggobi permet maintenant d'utiliser directement Gobi depuis R.

En python

J'ai essayé de faire en python ce que j'avais fait en R. Après plusieurs années d'efforts et de pratiques, je déconseille désormais formellement l'utilisation de python et de la bibliothèque bien connue. Celle-ci est très mal documentée et visiblement développée par des gens qui parlent de fouille de données sans la pratiquer. Il n'y a aucune pédagogie dans la documentation et la documentation n'est pas adaptée à des étudiants dans les cours de fouille de données en licence et en master. Il est totalement ridicule qu'à de nombreuses reprises, pour avoir de la documentation, il faut regarder le programme source : cela est contraire à tous les principes de bonne programmation : un programme source n'est pas une documentation, même s'il doit être commenté.

Je ne résiste pas à donner un exemple de ce qui me consterne avec python : les messages d'erreur incomépréhensibles. Admirez le travail de python :
